# stardewmod

Learning C# was quite easy, as the language is quite similar to Java, so there wasn’t much issue there. Some resources we used early on were W3Schools (https://www.w3schools.com/cs/index.php) and YouTube videos, such as Programming with Mosh’s C# tutorial (https://www.youtube.com/watch?v=gfkTfcpWqAY). Using SMAPI was quite easy as well, though we had to constantly refer back to the Stardew Valley Wiki (https://stardewvalleywiki.com/Modding:Index) and Stardew Modding Wiki (https://stardewmodding.wiki.gg/) in order to figure out what events would be appropriate for our saving function. Important classes, inherited interfaces, and functions that we used were ManualSaveData: the class we used to store the player’s state and OnButtonPress: the function we used to assign K as the save. We struggled at first with saving. Initially, we had thought we got the save feature to work, however we realized that the only thing it would save was the player’s position, and no stats would be updated. We figured out shortly after that we needed to account for way more than we expected, like stamina, health, money, and inventory. If we had more time, we would’ve tried to make it so that the player could assign their own keybind to save instead of K, and we had created an icon that would be used alongside the “Game Saved Successfully” message that appears in game, however, we would have to make our own type of message in order to get it to show up, so instead, we just used the error message as a placeholder. Overall, this experience was both fun and rewarding, since we got to learn SMAPI and C# alongside making a really useful mod that we both use in our game.
